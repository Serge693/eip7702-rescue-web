# EIP-7702 Rescue Web — Помощь (Русский)

## Что это?

Браузерный инструмент для спасения ETH и ERC-20 токенов со скомпрометированного кошелька без окна для перехвата атакующим. Использует EIP-7702 — новый стандарт Ethereum, позволяющий обычному кошельку (EOA) временно действовать как смарт-контракт.

---

## Что нужно перед началом

| Что | Зачем |
|-----|-------|
| **Source private key** | Приватный ключ скомпрометированного кошелька |
| **Sponsor private key** | Чистый кошелёк, который платит газ — нужно ≥ 0.005 ETH в целевой сети |
| **Destination address** | Безопасный адрес куда уйдут средства |

> **Sponsor кошелёк не должен быть скомпрометирован.** Используйте свежий кошелёк или аппаратный.

---

## Пошаговая инструкция

### Шаг 1 — Откройте приложение

Подключите безопасный кошелёк кнопкой **Connect Wallet** (верхний правый угол) — адрес автоматически подставится в поле Destination. Можно ввести адрес вручную.

---

### Шаг 2 — Заполните форму Config

**Source Private Key**
Приватный ключ скомпрометированного кошелька. Приложение выводит адрес ниже поля — проверьте что он совпадает.

**Sponsor Private Key**
Приватный ключ кошелька, который платит газ. Нужно достаточно ETH в выбранной сети (минимум 0.005 ETH проверяется автоматически).

**Destination Address**
Куда уйдут все спасённые средства. Дважды проверьте адрес — средства нельзя вернуть.

**Network**
Выберите сеть где скомпрометированный кошелёк держит средства. Поддерживается: **Base**, **Ethereum**.

---

### Шаг 3 — Claim & Токены (опционально)

Разверните эту секцию если нужно заклеймить токены перед rescue, или вывести конкретные ERC-20.

**Claim Contract + Claim Calldata**

Если есть незаклеймленные токены на наградной платформе (Layer3, Galxe и т.д.):

1. Откройте страницу клейма
2. Подключите **скомпрометированный** кошелёк
3. Нажмите кнопку клейма
4. В MetaMask откройте вкладку **Hex**
5. Скопируйте hex данные и запишите адрес **To**
6. **НЕ подтверждайте** транзакцию в MetaMask
7. Вставьте в приложение:
   - **Claim Contract** = адрес "To" из MetaMask
   - **Claim Calldata** = hex данные

> Подписи клейма истекают. Выполните rescue сразу после копирования.

**Claim Token**
Адрес ERC-20 токена который вы получите после клейма. Rescue контракт перешлёт его на destination.

**Дополнительные токены для rescue**
ERC-20 токены уже лежащие на source кошельке. Вставьте адреса через запятую:
```
0xTokenA,0xTokenB,0xTokenC
```

**Кастомный RPC URL**
Публичные RPC иногда не поддерживают симуляцию транзакций. Если видите "Simulation failed" — вставьте приватный RPC:
- Alchemy: `https://base-mainnet.g.alchemy.com/v2/YOUR_KEY`
- Infura: `https://mainnet.infura.io/v3/YOUR_KEY`

---

### Шаг 4 — Dry Run (рекомендуется в первый раз)

Включите **DRY RUN** перед выполнением. Симулирует rescue и показывает примерную стоимость газа без отправки транзакций. Если симуляция прошла — выключите Dry Run и выполните по-настоящему.

---

### Шаг 5 — Проверка

Нажмите **Проверить →**. Внимательно проверьте каждое поле:

- Source адрес совпадает со скомпрометированным кошельком
- Destination адрес правильный
- Сеть совпадает с той где лежат средства
- Режим показывает **LIVE** (или DRY RUN при тестировании)

---

### Шаг 6 — Выполнение

Нажмите **Выполнить Rescue →**. Приложение:

1. Проверит подключение к RPC
2. Проверит балансы source и sponsor
3. Проверит что rescue контракт задеплоен
4. Подпишет EIP-7702 авторизацию (без транзакции)
5. Отправит атомарную транзакцию: делегирование + rescue в одном выстреле
6. Дождётся подтверждения
7. (Если указаны rescue tokens) Отправит вторую tx для ERC-20 токенов
8. Отзовёт делегацию — кошелёк вернётся в обычный EOA
9. Покажет ссылки на транзакции в блокэксплорере

---

## После rescue

- ✅ Средства на destination адресе
- ✅ Делегация source кошелька отозвана
- ⚠️ **Прекратите использовать скомпрометированный кошелёк.** Атакующий по-прежнему имеет приватный ключ.
- ⚠️ Если sponsor кошелёк временный — переведите остатки на безопасный адрес.

## Устранение проблем

### "Simulation FAILED"

Публичный RPC не поддерживает `eth_call` с `stateOverride`. Два варианта:

1. Добавьте приватный RPC в поле **Кастомный RPC URL** (Alchemy или Infura)
2. Или продолжайте — если уверены в настройках, реальная транзакция может пройти

### "Sponsor balance too low"

Пополните sponsor кошелёк ETH в нужной сети. Нужно минимум 0.005 ETH для покрытия газа.

### "claim reverted with no reason"

Подпись calldata истекла. Вернитесь на сайт клейма, повторите шаги 1–7 из секции Claim, и немедленно выполните rescue.

### "RPC chain ID mismatch"

Указанный кастомный RPC URL предназначен для другой сети. Используйте правильный URL для выбранного чейна.

### Source кошелёк показывает 0 баланс, но у меня есть токены

Баланс ETH может быть нулевым при наличии ERC-20 токенов. Добавьте адреса токенов в **Дополнительные токены для rescue** — rescue выполнится.

## Частые вопросы

**Держит ли rescue контракт мои средства?**
Нет. Немедленно пересылает всё на destination адрес в той же транзакции.

**Можно ли переиспользовать rescue контракт?**
Да. Задеплоенные контракты на Base и Ethereum обрабатывают неограниченное количество rescue — один экземпляр на сеть, без повторного деплоя.

**Что если destination адрес неправильный?**
Средства нельзя вернуть. Всегда проверяйте перед выполнением.

**Есть ли временное ограничение?**
Только если используете claim calldata — такие подписи истекают (обычно через минуты или часы). В остальном временных ограничений нет.

**Можно ли сделать rescue из нескольких сетей одновременно?**
Веб-интерфейс работает с одной сетью за раз. Для мультисетевых rescue используйте CLI инструмент.
