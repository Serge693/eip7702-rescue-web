  EIP-7702 RESCUE WEB — HELP (ENGLISH)
═══════════════════════════════════════════════════════════════

WHAT IS THIS?
─────────────
A browser tool for rescuing ETH and ERC-20/ERC-721/ERC-1155
tokens from a compromised wallet. Uses EIP-7702 — an Ethereum
standard that lets a regular wallet (EOA) temporarily act as
a smart contract.

Delegation and rescue happen in ONE transaction — sweeper bots
have zero window to front-run.


WHAT YOU NEED
─────────────
  Source private key    Private key of the compromised wallet
  Sponsor private key   A clean wallet that pays gas (need ≥ 0.005 ETH)
  Destination address   Safe address where funds will be sent

  ! The sponsor wallet must NOT be compromised.
    Use a fresh wallet or a hardware wallet.


STEP-BY-STEP GUIDE
───────────────────

  STEP 1 — Open the app
  ──────────────────────
  Connect your safe wallet via the Connect Wallet button
  (top right) — the address auto-fills the Destination field.
  You can also type the destination manually.

  Language switch: EN / RU buttons in the header.


  STEP 2 — Fill in the form (section 01 / Wallets)
  ──────────────────────────────────────────────────
  Source Private Key
    The private key of the compromised wallet.
    The app shows the derived address below the field — verify it.

  Sponsor Private Key
    The private key of the wallet that pays gas.
    Must have enough ETH on the selected network (min 0.005 ETH).

  Destination Address
    Where all rescued funds will be sent.
    Double-check this address — funds cannot be recalled.


  STEP 3 — Select network (section 02 / Network)
  ────────────────────────────────────────────────
  Supported: Base, Ethereum.
  Select the network where the compromised wallet holds funds.


  STEP 4 — Claim & Tokens (section 03, optional)
  ────────────────────────────────────────────────
  Expand this section to claim tokens or rescue specific ERC-20s.

  Claim Contract + Claim Calldata
    If you have unclaimed tokens on a reward platform
    (Layer3, Galxe, etc.):

    1. Open the claim page
    2. Connect the COMPROMISED wallet
    3. Click the claim button
    4. In MetaMask open the Hex tab
    5. Copy the hex data and note the To address
    6. Do NOT confirm the MetaMask transaction
    7. Paste into the app:
         Claim Contract = the To address from MetaMask
         Claim Calldata = the hex data

    ! Claim signatures expire. Execute rescue immediately after copying.

  Claim Token
    Address of the ERC-20 token received after the claim.

  Additional Rescue Tokens
    ERC-20 tokens already held by the source wallet.
    Paste addresses separated by commas:
      0xTokenA,0xTokenB,0xTokenC

  Custom RPC URL
    Public RPCs sometimes don't support simulation.
    If you see "Simulation failed" — paste a private RPC:
      Alchemy : https://base-mainnet.g.alchemy.com/v2/YOUR_KEY
      Infura  : https://mainnet.infura.io/v3/YOUR_KEY


  STEP 5 — NFTs (section 04, optional)
  ──────────────────────────────────────
  To rescue NFTs, enter one token per line.

  ERC-721 format:
    0xContractAddress:TokenID
    Example: 0xBC4CA0EdA7647A8aB7C2061c2E118A18a936f13D:1234

  ERC-1155 format:
    0xContractAddress:TokenID:Amount
    Example: 0x60E4d786628Fea6478F785A6d7e704777c86a7c6:5678:2

  Multiple NFTs — one per line:
    0xBC4CA0EdA7647A8aB7C2061c2E118A18a936f13D:1234
    0x60E4d786628Fea6478F785A6d7e704777c86a7c6:5678:2

  Find the contract address on Etherscan / Basescan
  on the NFT page → Contract.


  STEP 6 — DRY RUN (recommended first time)
  ──────────────────────────────────────────
  Toggle DRY RUN before executing.
  Simulates the rescue and shows estimated gas cost
  without sending any transactions.
  If simulation passes — turn it off and execute for real.


  STEP 7 — Review
  ────────────────
  Click "Review →". Verify:
    · Source address matches the compromised wallet
    · Destination address is correct
    · Network matches where the funds are
    · Mode shows LIVE (or DRY RUN when testing)


  STEP 8 — Execute
  ─────────────────
  Click "Execute Rescue →". The app will:

    1. Check RPC connectivity
    2. Verify source and sponsor balances
    3. Verify rescue contract is deployed
    4. Sign EIP-7702 authorization (no transaction)
    5. Send atomic tx: delegation + rescue in one shot
    6. Wait for confirmation
    7. If ERC-20 tokens specified — send a second tx
    8. If NFTs specified — send a separate tx per NFT
    9. Revoke delegation — wallet restored to normal EOA
   10. Show transaction links to block explorer


AFTER RESCUE
────────────
  ✓ Funds are at the destination address
  ✓ Source wallet delegation has been revoked
  ! Stop using the compromised wallet — the attacker
    still has the private key
  ! If the sponsor wallet is temporary — move any
    remaining funds to a safe address


TROUBLESHOOTING
───────────────
  "Simulation FAILED"
    The public RPC doesn't support stateOverride.
    Add a private RPC in the Custom RPC URL field.

  "Sponsor balance too low"
    Top up the sponsor wallet with ETH (min 0.005 ETH).

  "claim reverted with no reason"
    The calldata signature expired. Go back to the claim site,
    get fresh calldata, and execute rescue immediately.

  "RPC chain ID mismatch"
    The custom RPC URL is for a different network.
    Use the correct URL for the selected chain.

  Source shows 0 ETH but I have tokens
    ETH balance can be zero while ERC-20/NFT tokens exist.
    Add token addresses — rescue will still execute.


FAQ
───
  Does the rescue contract hold my funds?
    No. It immediately forwards everything to destination
    in the same transaction.

  Can I reuse the rescue contract?
    Yes. One deployed instance per network handles
    unlimited rescues — no redeployment needed.

  What if the destination address is wrong?
    Funds cannot be recovered. Always verify before executing.

  Is there a time limit?
    Only if using claim calldata — those signatures expire.
    Everything else has no time limit.

  Can I rescue from multiple networks at once?
    The web UI does one network per rescue.
    Use the CLI tool for multi-network rescues.
