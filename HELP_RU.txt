  EIP-7702 RESCUE WEB — СПРАВКА (РУССКИЙ)
═══════════════════════════════════════════════════════════════

ЧТО ЭТО?
─────────
Браузерный инструмент для спасения ETH и ERC-20/ERC-721/ERC-1155
токенов со скомпрометированного кошелька. Использует EIP-7702 —
стандарт Ethereum, позволяющий обычному кошельку (EOA) временно
действовать как смарт-контракт.

Делегирование и rescue происходят в ОДНОЙ транзакции — у
sweeper-ботов нет окна для перехвата средств.


ЧТО НУЖНО ПЕРЕД НАЧАЛОМ
────────────────────────
  Source private key    Приватный ключ скомпрометированного кошелька
  Sponsor private key   Чистый кошелёк, платит газ (нужно ≥ 0.005 ETH)
  Destination address   Безопасный адрес куда уйдут средства

  ! Sponsor кошелёк не должен быть скомпрометирован.
    Используйте свежий кошелёк или аппаратный.


ПОШАГОВАЯ ИНСТРУКЦИЯ
─────────────────────

  ШАГ 1 — Откройте приложение
  ────────────────────────────
  Подключите безопасный кошелёк кнопкой Connect Wallet (верхний
  правый угол) — адрес автоматически подставится в поле
  Destination. Можно ввести адрес вручную.

  Переключение языка: кнопки EN / RU в хедере.


  ШАГ 2 — Заполните форму (секция 01 / Wallets)
  ────────────────────────────────────────────────
  Source Private Key
    Приватный ключ скомпрометированного кошелька.
    Приложение показывает адрес ниже поля — проверьте совпадение.

  Sponsor Private Key
    Приватный ключ кошелька который платит газ.
    Нужно достаточно ETH в выбранной сети (мин. 0.005 ETH).

  Destination Address
    Куда уйдут все спасённые средства.
    Дважды проверьте адрес — средства нельзя вернуть.


  ШАГ 3 — Выберите сеть (секция 02 / Network)
  ─────────────────────────────────────────────
  Поддерживаемые сети: Base, Ethereum.
  Выберите ту где скомпрометированный кошелёк держит средства.


  ШАГ 4 — Claim & Токены (секция 03, опционально)
  ─────────────────────────────────────────────────
  Разверните если нужно заклеймить токены или вывести ERC-20.

  Claim Contract + Claim Calldata
    Если есть незаклеймленные токены (Layer3, Galxe и т.д.):

    1. Откройте страницу клейма
    2. Подключите СКОМПРОМЕТИРОВАННЫЙ кошелёк
    3. Нажмите кнопку клейма
    4. В MetaMask откройте вкладку Hex
    5. Скопируйте hex данные и запишите адрес To
    6. НЕ подтверждайте транзакцию в MetaMask
    7. Вставьте в приложение:
         Claim Contract = адрес To из MetaMask
         Claim Calldata = hex данные

    ! Подписи клейма истекают. Выполните rescue сразу после копирования.

  Claim Token
    Адрес ERC-20 токена который получите после клейма.

  Дополнительные токены для rescue
    ERC-20 токены уже лежащие на source кошельке.
    Вставьте адреса через запятую:
      0xTokenA,0xTokenB,0xTokenC

  Кастомный RPC URL
    Публичные RPC иногда не поддерживают симуляцию.
    Если видите "Simulation failed" — вставьте приватный RPC:
      Alchemy : https://base-mainnet.g.alchemy.com/v2/YOUR_KEY
      Infura  : https://mainnet.infura.io/v3/YOUR_KEY


  ШАГ 5 — NFT (секция 04, опционально)
  ──────────────────────────────────────
  Для вывода NFT укажите один токен на строку.

  Формат ERC-721:
    0xАдресКонтракта:TokenID
    Пример: 0xBC4CA0EdA7647A8aB7C2061c2E118A18a936f13D:1234

  Формат ERC-1155:
    0xАдресКонтракта:TokenID:Количество
    Пример: 0x60E4d786628Fea6478F785A6d7e704777c86a7c6:5678:2

  Несколько NFT — каждый с новой строки:
    0xBC4CA0EdA7647A8aB7C2061c2E118A18a936f13D:1234
    0x60E4d786628Fea6478F785A6d7e704777c86a7c6:5678:2

  Адрес контракта можно найти на Etherscan / Basescan
  на странице NFT → Contract.


  ШАГ 6 — DRY RUN (рекомендуется в первый раз)
  ──────────────────────────────────────────────
  Включите переключатель DRY RUN перед выполнением.
  Симулирует rescue и показывает примерную стоимость газа
  без отправки транзакций.
  Если симуляция прошла — выключите и выполните по-настоящему.


  ШАГ 7 — Проверка
  ─────────────────
  Нажмите "Проверить →". Проверьте:
    · Source адрес совпадает со скомпрометированным кошельком
    · Destination адрес правильный
    · Сеть совпадает с той где лежат средства
    · Режим: LIVE (или DRY RUN при тестировании)


  ШАГ 8 — Выполнение
  ───────────────────
  Нажмите "Выполнить Rescue →". Приложение:

    1. Проверит подключение к RPC
    2. Проверит балансы source и sponsor
    3. Проверит что rescue контракт задеплоен
    4. Подпишет EIP-7702 авторизацию (без транзакции)
    5. Отправит атомарную tx: делегирование + rescue
    6. Дождётся подтверждения
    7. Если указаны ERC-20 токены — отправит вторую tx
    8. Если указаны NFT — отправит отдельную tx на каждый
    9. Отзовёт делегацию — кошелёк вернётся в обычный EOA
   10. Покажет ссылки на транзакции в блокэксплорере


ПОСЛЕ RESCUE
────────────
  ✓ Средства на destination адресе
  ✓ Делегация source кошелька отозвана
  ! Прекратите использовать скомпрометированный кошелёк —
    атакующий по-прежнему имеет приватный ключ
  ! Если sponsor кошелёк временный — переведите остатки
    на безопасный адрес


УСТРАНЕНИЕ ПРОБЛЕМ
──────────────────
  "Simulation FAILED"
    Публичный RPC не поддерживает stateOverride.
    Добавьте приватный RPC в поле Кастомный RPC URL.

  "Sponsor balance too low"
    Пополните sponsor кошелёк ETH в нужной сети (мин. 0.005 ETH).

  "claim reverted with no reason"
    Подпись calldata истекла. Вернитесь на сайт клейма,
    повторите шаги получения calldata и сразу выполните rescue.

  "RPC chain ID mismatch"
    Кастомный RPC URL предназначен для другой сети.
    Используйте правильный URL для выбранного чейна.

  Source показывает 0 ETH, но есть токены
    ETH баланс может быть нулевым при наличии ERC-20/NFT.
    Добавьте адреса токенов — rescue выполнится.


ЧАСТЫЕ ВОПРОСЫ
──────────────
  Держит ли rescue контракт мои средства?
    Нет. Немедленно пересылает всё на destination
    в той же транзакции.

  Можно ли переиспользовать rescue контракт?
    Да. Один экземпляр на сеть, без повторного деплоя.

  Что если destination адрес неправильный?
    Средства нельзя вернуть. Всегда проверяйте перед выполнением.

  Есть ли временное ограничение?
    Только если используете claim calldata — подписи истекают.
    В остальном временных ограничений нет.

  Можно ли rescue из нескольких сетей сразу?
    Веб UI работает с одной сетью за раз.
    Для мультисетевых rescue используйте CLI инструмент.
